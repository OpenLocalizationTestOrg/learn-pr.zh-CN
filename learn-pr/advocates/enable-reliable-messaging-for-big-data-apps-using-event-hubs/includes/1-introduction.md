大型数据应用程序应能够通过向之外扩展以满足增加的事务量来处理增加的吞吐量。

假设您在银行的信用卡部门工作。 你是管理负责欺诈测试的系统的团队的一部分, 以确定是批准还是拒绝每个事务。 系统接收到一个事务流, 并需要实时处理它们。

系统上的负载可能会在周末和假期期间高峰。 它应能够高效、准确地处理增加的吞吐量。 在考虑事务的敏感性质的情况下, 即使 slightest 错误也会产生重大影响。

Azure 事件中心可以接收和处理大量事务。 它还可以配置为在需要时动态扩展以处理增加的吞吐量。
在本模块中, 您将了解如何将事件中心连接到应用程序并可靠地处理大型事务量。

## <a name="learning-objectives"></a>学习目标

在本模块中, 您将:

- 使用 Azure CLI 创建事件中心。
- 将应用程序配置为通过事件中心发送或接收邮件。
- 使用 Azure 门户评估事件中心性能。
